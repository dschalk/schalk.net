Experiments with React, Websockets, and Monads

The code for these demonstrations is open source and availablpe at github.com/dschalk/

"JS-monads-part1" begins the Javascript Monads series. The previous publications were exploratory in nature and had various definitions for Monad and MonadIter.

"JS-monads-part2 demonstrates the handling of numeric and string data provided by a Haskell web server. The monads run the data through successive computations guided by user interaction with the browser. There is also a decision tree using MonadIter instances. What happens is guided by user computations of numbers from random throws of four dice. Most of the api is available for experimentation in the browser console.

Like Websockets Monads Part 2, Part 3 demonstrates guided iteration along a chain of monads and a tree with multiple branches. The tree has been expanded for more decision possiblities, and there is a score board for multi-player competition. "updateCalc", controls flow of action. Quite a bit of functionality is handled with not many lines of code. Click JS-monads-part3 for more information and a demonstration of the multi-player game.

Part 4 features the game from Part 3 using Motorcycle.js. It is a work in progress.

"javascript-monads" uses a deprecated version of Monad. It has an "fmap" method.

The game of Score features a Haskell Websockets server on the back end, and React.js on the front end. I did not use React.js as intended, and was on the verge of abandoning altogether. I found Snabbdom to much more effective in my applications. A lengthy explanation is at Game of Score
