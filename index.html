I experimented with various definitions of "Monad". The current state of the project can be found in JS-monads-part4 and JS-monads-part5. The Monad "ret" method no longer self-mutilates the calling monad by replacing its "x" attribute. Instead, it creates a new monad with the calling monad's name. We can create a monad named "mon" with value "valA" as follows: "var mon = mew Monad(valA, "mon"). "var a = mon" creates a reference to it. If we run "mon.ret(valB)" and "var b = mon", a will still be a monad with value valA and id 'mon' and b will be a monad with value valB and id 'mon'. References to "mon" point to the new monad. Had we not created a reference to the old monad "mon", it would be subject to garbage collection.
